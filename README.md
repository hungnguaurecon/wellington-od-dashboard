# OD Select Link Dashboard — Static Site

A single-page static dashboard for visualising TomTom Origin-Destination Select Link traffic flow data across the Wellington region. Designed for deployment on GitHub Pages (no server required).

## Architecture

```
static_site/
├── index.html          # Single-page app with 3 views (Index, Overview, Dashboard)
├── css/
│   └── theme.css       # Full theme: navy header, sidebar, controls, map styles
├── js/
│   ├── app.js          # Page routing, event wiring, initialisation
│   ├── map.js          # Leaflet map rendering, annotations, segment merging
│   ├── filters.js      # Cascading dropdown filters with metadata/manifest
│   └── symbology.js    # Sunset/plasma colour gradient and line weight scaling
├── data/               # Pre-exported GeoJSON files (generated by export_static.py)
│   ├── metadata.json   # Analysis metadata (route, location, lat/lon, folder)
│   ├── manifest.json   # Maps direction/folder → available time periods
│   ├── outgoing/       # GeoJSON files per analysis folder and time period
│   └── incoming/       # GeoJSON files per analysis folder and time period
└── export_static.py    # Python script to convert parquet → optimised GeoJSON
```

## JS Modules

### `app.js`
Entry point. Handles page navigation (Index/Overview/Dashboard), wires direction and annotation toggle buttons, initialises filters, and triggers map loads on selection changes. Displays the stat box with trip count (`api_trips_counted`) and formatted date range (`date_start`/`date_end`) from TomTom API metadata.

### `map.js`
Core map rendering module using Leaflet with Canvas renderer for performance. Key features:
- **Segment merging**: Adjacent LineString segments with the same rounded pct and direction are merged into continuous polylines for seamless rendering.
- **Dynamic annotations**: Zoom-adaptive labels (arrow + percentage) placed along lines using `L.divIcon` markers. Annotation density adjusts across 3 zoom tiers.
- **Annotation toggle**: On/Off control to show or hide percentage labels.
- **Overview map**: Separate Leaflet map with clustered markers for all analysis locations, lazy-initialised on first visit.

### `filters.js`
Cascading dropdown filter chain: Route → Location → Direction → Day of Week → Vehicle Type → Time Period. Loads `metadata.json` and `manifest.json` at init, rebuilds downstream options when any filter changes.

### `symbology.js`
Colour and weight calculations for traffic density visualisation:
- **Colour**: 11-stop plasma gradient (pale yellow → deep purple) interpolated by pct_trips.
- **Weight**: Linear scale from 1px (0%) to 14px (100%).
- **Legend**: CSS gradient string for the map legend control.

## Data Pipeline

### Metadata Enrichment

```bash
python fetch_link_metadata.py --with-api
```

Enriches `link_metadata.json` with data from the TomTom Analysis Preview API:
- `api_trips_counted`: Total trips recorded for the analysis
- `date_start` / `date_end`: Actual date range of the analysis (e.g., `2025-03-01` to `2025-03-31`)

These fields are displayed in the sidebar stat box as a formatted date range (e.g., "1 Mar 2025 – 31 Mar 2025").

### Prerequisites
```
pip install geopandas shapely
```

### Export
```bash
python export_static.py
```

Converts parquet data files into optimised GeoJSON:
1. Filters features with `pct_trips < 5%`
2. Simplifies geometry (tolerance 0.0001 ~ 10m)
3. Rounds coordinates to 5 decimal places
4. Rounds `pct_trips` to 1 decimal
5. Generates `metadata.json` and `manifest.json` index files

Output goes to `data/` directory.

## Running Locally

```bash
cd static_site
python -m http.server 8080
```

Open http://localhost:8080 in a browser.

## Deploying to GitHub Pages

1. Push the `static_site/` directory contents to a GitHub repository
2. In repository Settings → Pages, set source to the branch/folder containing `index.html`
3. The site will be available at `https://<username>.github.io/<repo>/`

## Key Technical Decisions

- **Canvas renderer** (`L.canvas()`): Renders 500+ polylines on a single `<canvas>` element instead of individual SVG paths, dramatically improving pan/zoom performance.
- **Client-side segment merging**: Adjacent segments with the same rounded pct_trips are merged into continuous polylines at load time, eliminating visible seams between segments.
- **Zoom-adaptive annotations**: Three annotation tiers (zoom 8-10, 11-13, 14+) control minimum pct threshold, marker spacing, and markers-per-line to keep the map readable at every zoom level.
- **Retina tiles with correct sizing**: CARTO `{r}` retina tiles use `tileSize: 512` and `zoomOffset: -1` to prevent sub-pixel white line artefacts between tile boundaries.
- **Pixel-space angle calculation**: Annotation rotation uses `latLngToContainerPoint` for Mercator-accurate alignment with road direction.
